## 1. 판단부 아키텍처 설계 v0.2 (`docs/decision_architecture.md`)

- 미션을 **3출력(경로 / 목표속도 / 정지요구)** 으로 환원, 중재는 속도=min·정지=OR.
- 확정 결정: 정적=Path Shifting, 동적=TTC 정지, 소스=어댑터 추상화, 출력=경로+목표속도+정지, **ADAS ACC 상시**.
- I/O 경계: pose=`/gps_utm_odom`, 출력=`/selected_path`+`/decision`. `/ctrl_cmd` 변환은 제어팀 스코프.

## 2. 장애물 입력 — `obstacle_info_publisher` (global_path_planner)

`/Object_topic`(morai_msgs/ObjectStatusList, MORAI GT) → 자차 **base_link 상대좌표(SI)** → `/obstacle_info`.
- 신규 메시지 `katri_msgs/ObstacleInfo`, `ObstacleInfoArray` (id·type·is_pedestrian·x_m·y_m·yaw_rad·speed_mps·width·length·distance).
- pose는 객체와 같은 프레임인 `/Ego_topic`(EgoVehicleStatus) 사용 → ENU/UTM 오프셋 없이 상대변환.
- 단위 변환: 객체 velocity km/h→m/s, heading deg→rad. **MORAI size 규약 = (length, width, height)** (msg 주석 "(width,length,height)"는 오류 — 라이브 실측으로 교정).
- RViz: `/obstacle_markers`(MarkerArray) 타입별 모양·색 — 보행자=빨강 원기둥, NPC=파랑 큐브, 정적=주황 큐브.
- 테스트: `scripts/test_obstacle_info.py` (좌표/단위/타입→모양, 격리 마스터 PASS).

## 3. ADAS ACC — `acc_node` (global_path_planner)

`/obstacle_info` + `/Ego_topic` → `/acc/target_speed`[km/h]. 상시 동작.
- CTG(정속 시간간격) 선형 ACC: `v = lead_speed + k*(gap - (min_gap + τ*v_ego))`, `[0, v_set]` 클램프.
- 선행차 = 자차 차로(base_link y-게이트) 최근접 전방차. 없으면 v_set(60kph) 발행.
- 파라미터 전부 튜닝 노브(v_set/τ/min_gap/k/lane_gate/standoff).
- 테스트: `scripts/test_acc.py` (무선행차→60, gap일치→36, 근접→10.08, 옆차로→60, 격리 PASS).
- 한계(ponytail): y-게이트는 직선도로 근사 → ReferencePath(Frenet) 붙으면 교체.

## 4. 제어 브링업 — purepursuit 통합 + `adas_drive.launch`

무장애물 60kph 안정 주행이 목표. 샘플 제어기 `hybrid_curv_real`(purepursuit)로 경로 추종.
- 제어기 수정: `/acc/target_speed` 구독 → `velocity = min(내부속도, acc_cap)` (설계 "속도=min 중재"와 동일). ADAS가 속도 상한을 쥠.
- 제어 타입 = **type2(velocity)** 유지(임시). 규정 2-8은 type1(accel/brake) 요구 → 대회 전 전환 필요.
- `morai_velocity_scale=0.06` → 1000*0.06=60kph, 곡률 페널티로 커브 감속.
- purepursuit `ackermann_msgs`(미사용) 의존 제거해 빌드.
- 제어기 wheelbase는 원래값 1.04m 유지(아이오닉 3.0 튜닝은 보류).
- `launch/adas_drive.launch`: ll2utm→/gps_utm_odom, 2026 전역경로→/global_path1→odom_path→/local_path1(base_link), acc_node, 제어기, control_udp. **ll2utm 오프셋 east=302595.0 / north=4124145.0** (전역경로 로컬 프레임 정합, 라이브 검증: /gps_utm_odom≈경로 시작점).
