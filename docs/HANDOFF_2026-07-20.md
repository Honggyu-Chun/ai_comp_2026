## 3. 데이터 흐름 (판단→제어 파이프라인)

```
측위:  GPS → ll2utm(/gps_utm_odom) → gps_imu_fusion(EKF) → /odometry/filtered
인지:  /Object_topic → obstacle_info_publisher → /obstacle_info (base_link 상대)

경로:  전역경로 → odom_path_publisher → /local_path1 (base_link)
회피:      /local_path1 + /obstacle_info → static_obstacle_avoider → /avoid_path
선택:      local_path_selector → /selected_path
로터리:    /odometry/filtered + /obstacle_info → rotary_entry_node
               → /rotary/stop_wall_s, /rotary/speed_cap_kph
속도:      /selected_path + /obstacle_info + rotary → speed_profiler_node
               → /selected_path_profiled  (pose.position.z = 목표속도[kph])
제어:  /selected_path_profiled → mpc_control (z를 목표속도로 순수 추종)
```

**설계 원칙**: 판단이 경로 + **점별 목표속도(kph, Path.z)** + 정지요구를 만들고, 제어는 **순수 추종**만 한다. 속도 프로파일링은 전부 판단부(speed_profiler)로 이관됨. 중재: 속도=min, 정지=OR.

## 5. 로터리 심층 (가장 최근·복잡 — 필독)

### 5.1 FSM (`gpp::RotaryGate::update`)
```
FAR → APPROACH → { COMMIT(무정지 진입) | HOLD(정지 후 대기) } → INSIDE → DONE
```
- **활성/판정은 전부 "진입점까지 유클리드 거리(`dist_to_entry`)"** 로. (전방투영 s는 heading에 민감해 멀리서 수백 m 요동 → 게이팅에 쓰면 안 됨. s는 정지벽 위치·타이밍에만.)
- **FAR→APPROACH**: `dist_to_entry ≤ activate_dist_m(40)`.
- **APPROACH**: 매 프레임 `evaluateGap`(TTC). `dist_to_entry > activate×1.5`면 FAR 복귀(히스테리시스). 갭 연속 열림(`go_confirm_frames`) **且 `dist_to_entry ≤ commit_dist_m(20)`** 이면 COMMIT. 정지선 도달(`dist ≤ s_stopline+1`)했는데 미확정이면 HOLD. 그 외엔 정지벽+접근속도.
- **COMMIT**: 정지벽 절대 재생성 안 함(래치). cap=`approach_cap_kph`(아직 링 밖 접근직선). `ego_in_ring`(물리적 링 진입) 시 INSIDE.
- **INSIDE**: cap=`v_ring_kph`(20, 링 내부). 진출각까지 각도진행 완료 **또는** 진행 후 링 밖으로 나감 → DONE.
- **DONE**: `allow_rearm=false`면 영구 래치. true면 진입점서 멀어지면(`dist>activate`) reset.

### 5.2 TTC 갭수용 (`evaluateGap`)
- 순환차 필터: `isRingAgent`(|r-R|<`ring_band` && speed>`v_agent_min`) + 방향(`cross(p-C,v)` 부호 == `dir`).
- 각 순환차 진입점 도달시각 `eta = wrap2π(dir·(θ_entry-θ_i))/ω`. 자차 도착창 `[t_arrive-τ_lead, t_clear+τ_lag]` 침범하면 거부.
- 자차 도달/통과: 사다리꼴 발진 모델(`egoArriveClear`).
- 데드락: HOLD가 `deadlock_timeout_s` 넘으면 τ를 `relaxed_scale`로 1회 완화.

### 5.3 캘리브레이션 (기하는 반드시 실측)
```bash
rosrun global_path_planner fit_rotary_circle.py _duration:=15
# /obstacle_info + /odometry/filtered 로 순환NPC 월드복원 → RANSAC+Kåsa 원피팅
# → center_x/y, ring_radius_m, dir 출력. launch 에 반영.
```
- theta_entry/exit: 전역경로가 링에 합류/이탈하는 **UTM 좌표**로 `atan2(y-cy, x-cx)` 계산.
- **현재 확정값(K-City, UTM)**: center=(-101.5695, 343.7351), R=18.0241, dir=1(CCW), θ_entry=-1.1460, θ_exit=-0.4611. (MORAI 월드와 <0.5m 차이 — K-City는 MORAI월드≈UTM.)

### 5.4 디버깅 로그 읽는 법
```
[rotary_entry] phase=N s_conf=.. stop=.. cap=.. eta=.. block=<id> ring=<링안차수> all=<전체장애물>
```
- `phase`: 0=FAR 1=APPROACH 2=HOLD 3=COMMIT 4=INSIDE 5=DONE.
- `ring`: **링 안 순환차 수**(≠ all). 실제 로터리 도는 4대와 맞아야 함. 안 맞으면 기하(center/R) 오설정.
- `block`/`eta`: TTC가 막는 차 id와 도달시각. 이게 뜨면 TTC 정상 동작.
- RViz `/rotary/debug_markers`(base_link): 링·진입(노랑)/진출(파랑)·순환차(초록, 막는차 빨강)·상태텍스트·정지벽(빨강 CUBE).
